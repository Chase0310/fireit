# 🔥 fireit

### 开火 · Fire. Pass. Ship.

**有性格的多 agent 协作厨房 —— 一群有名字的厨师，对着共享的出菜口，把任务一道道端出来。**

*No head chef. Just a brigade that fires on call.*

</div>

---

## 这是什么

`fireit`（开火）是一个**多 agent 协作系统**，灵感来自专业厨房的 brigade（班底）协作模式。

它不是又一个"调 LLM 的框架"。它是让一群**有名字、有性格、有专长**的 AI agent，像厨房班底一样，
在一个共享出菜口（pass）前，**有节奏地接力、互审、出菜**——没有指挥一切的行政总厨（head chef / boss），
只有纪律护栏（hard rails）和让每个厨师自主发挥的软权力（soft power）。

> 厨房是地球上最古老、最高效的 peer 协作现场：忙而不乱，分工明确，靠黑话（"Fire!" "Pass!" "Behind!"）协调，
> 出问题靠喊（escalate），翻车了重做（retry）。`fireit` 把这套模式搬进 agent 协作。

---

## 为什么是厨房

因为厨房的黑话，和我们要做的东西是 **1:1 映射**：

| 厨房黑话 | 含义 | 在 fireit 里 | 对应概念 |
|---------|------|------------|---------|
| 🔥 **"Fire!"** | "开做这道菜！" | 一只厨师认领任务、开火 | 任务开始 |
| **"Pass!"** | 出菜口端出完成的菜 | 厨师宣布这步做完了 | 阶段完成 |
| **"86"** | 这道菜取消了 | 跳过某个步骤 | 阶段跳过 |
| **"Behind!"** | "端着热菜从你身后过！" | 厨师间互相协调 | agent 间通信 |
| **"In the weeds"** | 忙疯了 / 跟不上了 | 厨师报告卡住了 | 阶段阻塞 |
| **"On the fly"** | 加急单 | 紧急上报 | 优先级 / 升级 |
| **"Mise"** (mise en place) | 备料就位 | 设计确认 / 准备阶段 | 设计门 |
| 烧焦了 → 重做 | 😂 | 只重做烧焦那道菜 | 步骤级 retry |

> **"Fire!" 是一声号令，不是静态的工具名。** `fireit` 是一个动词——你喊一声，厨房就动起来。

## 设计哲学（5 条，来自 Raft + 厨房）

`fireit` 的每个设计决策都受这 5 条哲学约束（详见 `docs/research/raft-design-philosophy.md`）：

1. **厨师是持续身份，不是一次调用** — 每个厨师跨任务在场，有自己的手艺和记忆
2. **厨师有名字** — 名字 = 路由地址 + 口碑历史 + 信任锚点（不是匿名 worker）
3. **沉默也是协作** — 好厨师知道什么时候该闭嘴专心颠勺，什么时候该喊 "Behind!"
4. **纪律是地平线，不是天花板** — Hard Rails（卫生底线不可破）+ Soft Power（地平线之上自主发挥）
5. **当用户是厨师时，UX 变成 AX** — 厨师间的交接（"出菜口递菜"）要可直接行动，不能让对方猜

**一句话内核**：

> *把 agent 当"持续在场的、有名字的、懂得克制的队友"，用纪律护栏（而非行政总厨）托住底线，
> 用地平线之上的自主空间释放协作涌现。*

## 核心：无 Boss 的 Brigade（班底）

`fireit` 的核心机制是 **Feature 推进图（FPP）**——一个顶层、有状态、推进任务的协调结构，
但**没有 head chef**。它不是一个 boss 在分配工作，而是整个班底共享的一块出菜板（pass board）：

- **谁开火** — 任何厨师都能喊 "Fire!" 认领任务（事件驱动，非分配）
- **谁接班** — 仍靠 @mention 的"接/退/升"（join/peel/escalate），厨师自己决定
- **怎么 retry** — 烧焦了就重做那道菜，不影响其他菜（步骤级 retry）
- **牵头 ≠ 统治** — 领班厨师（lead cook）只是"当前在前面的人"，累了就换，不是指挥官

> 灵感来自爵士乐的**曲式（chord chart）**——顶层共享结构让即兴不变噪音，但曲式不规定每个音符，
> 没有任何人是指挥。`fireit` 是曲式，不是指挥。
>
> 详见 `docs/research/fpp-design-spec.md`。

## 继承的调研（`docs/research/`）

`fireit` 站在四个项目的肩膀上：

| 来源 | 借鉴了什么 |
|------|-----------|

| **Raft** | 5 条设计哲学（持续身份/名字/克制/纪律/AX） |
